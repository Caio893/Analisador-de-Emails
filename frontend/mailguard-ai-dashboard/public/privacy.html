Política de Privacidade

Esta política explica como o Email Radar coleta, usa, armazena, compartilha e protege dados quando o usuário utiliza o aplicativo para analisar e-mails por demanda e identificar possíveis riscos de spam, phishing, golpes ou conteúdo malicioso.

1. Quem somos

O Email Radar é um serviço de análise de segurança de e-mails. O aplicativo permite que o usuário conecte sua conta Google com acesso somente leitura ao Gmail e solicite uma avaliação automatizada de risco. O objetivo é ajudar o usuário a entender se um e-mail pode conter sinais de spam, phishing, tentativa de golpe, engenharia social, links suspeitos, anexos suspeitos ou outros indícios de ameaça.

Esta política se aplica ao site, ao aplicativo web e aos recursos relacionados do Email Radar disponíveis no domínio email-radar.com.

2. Dados que podemos coletar

Coletamos somente os dados necessários para fornecer as funcionalidades visíveis ao usuário. Dependendo de como o usuário utiliza o serviço, podemos processar:

  • Dados de conta: nome, endereço de e-mail, identificador de conta Google e informações básicas de perfil necessárias para autenticação.
  • Dados de e-mails sincronizados ou analisados pelo usuário: identificadores de mensagens e threads, marcadores ou pastas, remetente, destinatários, assunto, data, cabeçalhos técnicos, presença de anexos, nomes e tipos de anexos, links presentes no corpo da mensagem e trecho textual limitado do corpo do e-mail, quando necessário para análise.
  • Conteúdo limitado para análise: o Email Radar pode extrair parte do texto do e-mail, com limite operacional definido pelo aplicativo, para gerar a avaliação de segurança solicitada pelo usuário.
  • Dados técnicos: endereço IP, tipo de navegador, sistema operacional, data e hora de acesso, logs de erro, eventos de autenticação, uso de API e métricas operacionais.
  • Resultados da análise: classificação de risco, justificativas, alertas, recomendações de cautela e histórico de análises solicitadas, quando esse recurso estiver habilitado para o usuário.

O Email Radar solicita apenas os escopos necessários para autenticação e análise de segurança de e-mails. Atualmente, o aplicativo usa acesso Gmail somente leitura e não solicita permissão para enviar, editar, mover ou apagar e-mails.

3. Como usamos os dados

Usamos os dados coletados para:

  • autenticar o usuário e permitir acesso seguro ao aplicativo;
  • buscar ou receber os e-mails que o usuário escolheu analisar;
  • identificar sinais de spam, phishing, golpe, fraude, malware, links suspeitos, domínios enganosos ou anexos potencialmente perigosos;
  • gerar uma resposta de análise de risco, incluindo explicações e recomendações de segurança;
  • manter a segurança, prevenir abuso, investigar falhas e proteger a integridade do serviço;
  • cumprir obrigações legais e responder a solicitações legítimas de autoridades competentes, quando aplicável;
  • melhorar recursos visíveis ao usuário, desde que isso respeite as limitações desta política e as permissões concedidas.

4. Uso de dados recebidos das APIs do Google

Quando o usuário conecta uma conta Google, o Email Radar pode solicitar permissões por OAuth para acessar dados necessários ao funcionamento do serviço. O acesso é usado apenas para recursos de segurança de e-mail que o usuário vê e solicita dentro do aplicativo.

Limited Use disclosure: Email Radar's use and transfer to any other app of information received from Google APIs will adhere to Google API Services User Data Policy, including the Limited Use requirements.

The use of information received from Google Workspace APIs will adhere to the Google User Data Policy, including the Limited Use requirements.

Em português: o uso e a transferência, pelo Email Radar, de informações recebidas das APIs do Google seguirão a Política de Dados do Usuário dos Serviços de API do Google, incluindo os requisitos de Uso Limitado.

4.1. Dados do Gmail

Se o usuário autorizar acesso ao Gmail, o Email Radar poderá processar metadados e conteúdo de mensagens sincronizadas ou analisadas dentro do aplicativo para análise de segurança. Isso pode incluir remetente, assunto, cabeçalhos, snippets, links, marcadores, informação sobre anexos e trechos do corpo do e-mail necessários para classificar risco.

4.2. Limitação de finalidade

Dados obtidos via APIs do Google são usados somente para fornecer ou melhorar funcionalidades de segurança de e-mail visíveis ao usuário, como análise de spam, phishing e mensagens maliciosas. Não usamos esses dados para publicidade, perfilamento publicitário, venda, corretagem de dados, avaliação de crédito ou finalidades incompatíveis com o serviço.

4.3. Processamento por IA e treinamento de modelos

O Email Radar pode usar processamento automatizado e provedores de IA para classificar risco e gerar explicações. Enviamos apenas o mínimo necessário para a análise solicitada, como remetente, domínio do remetente, assunto, pasta e trecho textual limitado. Não enviamos tokens OAuth, refresh tokens, exportações brutas de caixa postal ou anexos brutos para provedores de IA.

O Email Radar não usa dados de e-mails, dados do Gmail ou dados recebidos das APIs do Google para criar, treinar ou melhorar modelos gerais de inteligência artificial ou machine learning. Quando uma análise automatizada é necessária, os dados são processados somente para gerar ou melhorar a resposta de segurança solicitada pelo usuário naquele contexto.

4.4. Acesso humano

Não permitimos que pessoas leiam o conteúdo dos e-mails do usuário, exceto quando houver consentimento explícito do usuário para suporte, quando for necessário para segurança e investigação de abuso, ou quando exigido por lei.

4.5. Transferência de dados

Não vendemos, alugamos ou transferimos dados do Google a anunciantes, data brokers, revendedores de informação ou plataformas de publicidade. Podemos transferir dados a prestadores de serviço estritamente necessários para hospedar, proteger, processar e entregar as funcionalidades do Email Radar, sempre com finalidade limitada ao serviço solicitado pelo usuário.

5. Compartilhamento com terceiros

Podemos compartilhar dados somente nas seguintes situações:

  • Provedores de infraestrutura: hospedagem, banco de dados, armazenamento, filas, logs, monitoramento e segurança.
  • Provedores de processamento de IA ou análise: quando necessário para gerar a avaliação de risco solicitada pelo usuário, com transmissão limitada ao conteúdo necessário para a análise.
  • Segurança e prevenção de abuso: para detectar, prevenir ou investigar fraude, abuso, falhas técnicas ou incidentes de segurança.
  • Obrigação legal: quando exigido por lei, ordem judicial, autoridade competente ou para proteger direitos legais.
  • Operações societárias: em caso de fusão, aquisição ou venda de ativos, com aviso e consentimento quando exigido pelas políticas aplicáveis e pela lei.

Não compartilhamos dados pessoais ou dados de e-mail para publicidade comportamental, revenda, enriquecimento de bases de terceiros ou criação de perfis comerciais.

6. Armazenamento e retenção

O Email Radar mantém dados pelo tempo necessário para fornecer o serviço, cumprir obrigações legais, manter segurança, resolver disputas e melhorar funcionalidades permitidas. Conteúdo de e-mail usado para análise deve ser mantido pelo menor tempo necessário para gerar a resposta, exibir o resultado ao usuário e manter registros operacionais essenciais.

Tokens de acesso e atualização OAuth, quando utilizados, são armazenados criptografados em repouso no backend do Email Radar e podem ser revogados pelo usuário a qualquer momento nas configurações da conta Google em https://myaccount.google.com/permissions. O aplicativo também pode enviar uma solicitação de revogação do token OAuth pela página de perfil quando a conta estiver conectada.

O usuário pode solicitar exclusão de conta, histórico de análises e dados associados conforme descrito nesta política. Após uma solicitação confirmada de exclusão, apagaremos os dados ativos associados à conta em até 30 dias, salvo quando retenção adicional for exigida por lei, segurança, prevenção de abuso ou resolução de disputas. Backups podem reter dados por até 90 dias antes da expiração.

7. Segurança

Adotamos medidas técnicas e organizacionais razoáveis para proteger dados contra acesso não autorizado, perda, alteração, divulgação ou destruição. Essas medidas podem incluir:

  • uso de HTTPS/TLS para transmissão de dados;
  • proteção de tokens, credenciais e segredos de API;
  • controles de acesso internos;
  • monitoramento de falhas, abuso e eventos de segurança;
  • limitação de coleta e retenção de dados;
  • separação entre dados operacionais e dados de análise quando aplicável.

Nenhum sistema é totalmente imune a riscos. Caso identifiquemos incidente que afete dados pessoais, adotaremos medidas de resposta e comunicação conforme a legislação aplicável.

8. Direitos do usuário e exclusão de dados

O usuário pode solicitar:

  • acesso aos dados pessoais associados à conta;
  • correção de dados incompletos ou desatualizados;
  • exclusão de conta, histórico de análises e dados associados;
  • revogação de consentimento e desconexão da conta Google;
  • informações sobre compartilhamento e tratamento de dados;
  • portabilidade, quando aplicável pela legislação.

O usuário conectado pode excluir dados locais ativos pela página de perfil do aplicativo. Para solicitar exclusão em backups, confirmar exclusão de conta ou exercer direitos de privacidade, envie uma mensagem para suporte@email-radar.com. Também é possível revogar o acesso do Email Radar pela página de permissões da Conta Google.

9. Cookies e tecnologias similares

Podemos usar cookies ou armazenamento local para manter sessão, lembrar preferências, proteger o login e melhorar a estabilidade do aplicativo. Não usamos cookies para vender dados pessoais ou para publicidade comportamental baseada em dados do Gmail.

10. Crianças e adolescentes

O Email Radar não é direcionado a crianças. O serviço deve ser usado por pessoas com idade suficiente para consentir com o tratamento de dados conforme a legislação aplicável ou com autorização de responsável legal.

11. Transferências internacionais

Provedores de hospedagem, infraestrutura ou processamento podem operar em outros países. Quando houver transferência internacional de dados, adotaremos medidas razoáveis para proteger as informações conforme esta política e a legislação aplicável.

12. Alterações nesta política

Podemos atualizar esta Política de Privacidade para refletir mudanças legais, técnicas ou operacionais. Quando alterações relevantes afetarem o uso de dados do Google ou dados pessoais, atualizaremos esta página e, quando necessário, solicitaremos novo consentimento do usuário antes de usar os dados para nova finalidade.

13. Contato

Para dúvidas sobre privacidade, segurança, exclusão de dados ou uso das APIs do Google, entre em contato pelo e-mail: suporte@email-radar.com.
